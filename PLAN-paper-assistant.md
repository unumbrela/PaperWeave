# 论文助手重定位大纲

> 目标：把当前项目从“分散的小工具集合”重构成一个围绕研究工作流展开的论文助手。
> 核心不是多做几个工具，而是把现有能力重新组织成一条清晰主线。

---

## 1. 新定位

### 一句话定位

`一个把读文献、做验证、写论文、讲结果串成同一条工作流的研究型论文助手。`

### 项目不再是什么

- 不再是泛化的 `AI Toolbox`
- 不再强调“我这里有很多工具”
- 不再按 `写作 / 编程 / 效率 / 学习` 这种工具分类组织首页

### 项目要成为什么

- 一个面向研究场景的工作流产品
- 一个从想法到表达的论文流程助手
- 一个既能辅助执行，也能辅助讲述的研究界面

### 核心价值

- 把零散任务收回到统一流程里
- 把抽象研究步骤变成可执行模块
- 把结果表达从“写出来”提升到“讲清楚、展示清楚”

---

## 2. 新名字

### 推荐名称

`PaperWeave`

### 推荐理由

- `Paper` 直接指向论文与研究场景
- `Weave` 强调把 idea、实验、写作、可视化重新编织成一条线
- 比 `Toolbox` 更聚焦，也比单纯的 `Paper Assistant` 更像产品名

### 推荐副标题

`PaperWeave · 研究型论文助手`

### 备选名称

- `PaperFlow`
  适合强调流程感，直观，但略通用
- `ResearchCanvas`
  适合强调“构思与表达”，更偏作品集气质
- `PaperForge`
  适合强调“从 idea 到论文落地”，风格更强，但语义更硬

### 命名建议

- 对外统一使用 `PaperWeave`
- 页面标题、README、副标题中保留中文解释：`研究型论文助手`

## 3. 新主线

建议把当前项目的叙事从“工具集合”改成下面这条研究流程：

`读文献 -> 生 idea -> 做验证 -> 写论文 -> 讲结果 -> 做可视化表达`

查论文

组织

### 3.1 读文献

目标：把外部资料快速转成可读、可摘取、可整理的研究输入。

对应能力：

- 网页摘要
- PDF / Word / HTML / TXT 转 Markdown
- 结构化提取关键点、引文、方法、实验设置

### 3.2 生 idea

目标：在已有文献和研究背景上生成更可做、可验证的研究方向。

对应能力：

- idea 生成器
- 研究空白识别
- 创新点假设
- 最小可行实验设计

### 3.3 做验证

目标：把一个模糊想法拆成真正能执行的研究任务链。

对应能力：

- 任务拆解器
- 实验规划
- skill 自动化封装
- 重复任务脚本化

### 3.4 写论文

目标：把代码、实验和结论转成结构清晰的论文内容。

对应能力：

- 方法流程梳理
- 模块解释
- 实验结果总结
- 论文大纲生成
- 图表与 caption 辅助

### 3.5 讲结果

目标：把项目讲给导师、同学、评审或答辩听众。

以其他外行人的角度

改写成经典模型的展示



### 3.6 可视化表达



目标：把研究结果变得更直观、更易传播。

对应能力：

- 模型流程可视化
- 特征图可视化
- embedding 降维可视化
- 数据集分布 / 聚类关系图
- 类似 iGEM 星云图的叙事型可视化



迁移应用：每个点的颜色 对应不同深度等等

此外，还做一些

即插即用

## 4. 产品结构重组

建议未来首页和导航都围绕流程组织，而不是围绕“工具类型”组织。

### 一级结构建议

- `Workflow`
- `Cases`
- `Visual Lab`
- `About`

### Workflow 下的 5 个主模块

- `Read`
  文献阅读与资料整理
- `Ideate`
  idea 生成与研究假设
- `Execute`
  任务拆解、实验推进、自动化
- `Write`
  论文撰写与项目讲述
- `Visualize`
  结果表达与交互可视化

### Cases 的作用

- 放你现有的 `CNN`、`医学图像分割` 等案例
- 不再把它们当作孤立 demo
- 而是作为“PaperWeave 如何讲清论文”的样板

### Visual Lab 的作用

- 放动效、背景、网页复刻、可视化实验
- 这些能力不应占据主产品心智
- 但可以作为未来论文展示模板与交互表达资产库

## 6. 缺失模块清单

下面是当前项目要真正成为论文助手，还缺少的关键能力。

### 6.1 必须补齐的核心模块

#### A. Idea 生成器

作用：从研究方向、关键词、约束条件出发，生成可做的研究 idea。

建议输入：

- 研究领域
- 目标问题
- 已知 baseline / 参考论文
- 可用数据集
- 资源限制

建议输出：

- 研究问题定义
- 创新假设
- 与 baseline 的差异点
- 最小验证实验
- 风险与失败点

#### B. Related Work 梳理器

作用：把若干篇论文或摘要整理成研究脉络。

建议输出：

- 研究主题聚类
- 方法演化线
- 常见实验设置
- 未被解决的空白点

#### C. 实验规划器

作用：把一个 idea 变成实验计划，而不是停留在概念层。

建议输出：

- baseline 列表
- 数据集与划分
- 指标
- ablation 清单
- 实验优先级
- 时间规划

#### D. 论文大纲生成器

作用：把已有代码与实验结果组织成论文骨架。

建议输出：

- 标题候选
- Abstract 草稿
- Introduction 逻辑
- Method 结构
- Experiment 章节结构
- Conclusion 要点

#### E. Storyline 生成器

作用：帮助你把项目讲给别人听，而不是只给出论文正文。

建议输出：

- 2 分钟介绍版本
- 5 分钟答辩版本
- 方法图讲解顺序
- 核心贡献点
- 常见问答准备

### 6.2 高价值增强模块

#### F. Figure Planner

作用：自动规划论文里该放哪些图。

建议输出：

- 方法总览图
- 模块细节图
- 结果对比图
- ablation 图
- 可视化案例图

#### G. Caption / Table Writer

作用：根据实验结果生成表格标题、图注、结果说明。

#### H. Rebuttal Assistant

作用：根据 reviewer comment 生成 rebuttal 草稿与补实验建议。

#### I. Dataset / Embedding Visualizer

作用：对点集、样本分布、特征 embedding 做降维与交互展示。

#### J. Paper Demo Builder

作用：把论文结果自动转成网页展示页或交互 demo 草稿。

## 7. 模块优先级

建议不要一下子补齐所有能力，而是先做闭环。

### Phase 1：先做最小论文闭环

包含模块：

- 文献网页速读器
- 论文资料结构化整理器
- Idea 生成器
- 研究任务规划器
- 研究自动化封装器
- 论文大纲生成器

### Phase 2：补上“讲清楚”的能力

包含模块：

- 方法实现解读器
- Storyline 生成器
- Figure Planner
- Caption / Table Writer

### Phase 3：补上“展示层”

包含模块：

- Dataset / Embedding Visualizer
- 论文交互讲解案例页
- Paper Demo Builder
- 科研项目叙事模板

## 12. 最终目标形态

如果这个重定位成功，`PaperWeave` 应该给人的感觉是：

- 它不是一个“顺手做了几个 AI 工具”的网站
- 它也不是一个单纯的“论文润色器”
- 它是一个服务研究流程的界面系统

用户进入后应该能明确感受到三件事：

- 我可以在这里读懂资料
- 我可以在这里推进研究
- 我可以在这里把研究讲清楚

---

如果只做三件事，建议是：
